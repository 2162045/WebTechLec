Specificity

The more specific you are in targeting the element by using different selectors the more likely it would be following the style implemented. Inline styles such as italic or bold will override styles from external style sheets. Universal selectors do not have any effect on specificity including combinators like (eg. >, ~). The !important method overrides all declarations. This method may be effective but a bad practice since it would be more difficult during the debugging process. Specificity will be applied to the element with multiple declarations and will take precedence based on the value of the selectors.

Origin of Style Sheets

CSS' algorithm is to determine CSS declarations in order to choose the appropriate values for CSS properties.

author – specifies style sheet for a source document according to the conventions of the document language.

user – the user may specify a file that contains a style sheet or the user agent may provide an interface that generates a user style sheet.

user agent – A user agent's default style sheet should present the elements of the document language in ways that satisfy general presentation expectations.

Calculating CSS Specificity Value

The browser calculates each selectors' specificity value and then compares them to check which among them is the highest and applies the declarations to the element targeted

Highest Specificity → Lowest Specificity

Style Attribute → ID → Class, pseudo-class, attribute → Elements
