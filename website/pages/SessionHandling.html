Session Handling

A session can be defined as the communication between two people. In web application session is the communication between the users and the servers in the form of the request and respond. It is a sequence of network HTTP request transaction associated with one user.
Sessions is a way of storing data for individual users contrary to a unique session ID. You can also invalidate the session by calling the session.invalidate() method

Many of us now uses internet and visits websites. Cookies

are also used in session management. It is stored in the users browser and sends back to the server if needed. It is a small piece of information that has a name, a single value and some attributes. Its life span depends on how long the users browser is running and by the time the users browser is closed all cookies will be destroyed. And when the user open the browser againit sends request to the server, and a new session is being created.

There are two Types of Cookies

Session Cookies

It lives in the user browsers memory and when the user browser is closed the cookies gets destroyed.

Persistent Cookies

It is destroyed based on its expiration date and it was stored in the users hard drive.

Sending Cookies to client

  1. Create a Cookie object
  2. Set the maximum Age
  3. Place the Cookie in HTTP response header
Disadvantage of using a cookies in session handling. Using cookies in session tracking is a bad practice because of the following:
  1. it will not availablle if ever the clients browser was cleared or disabled yhe cookies.
  2. It is more difficult to implement cookies for session tracking than other session management mechanism.
  3. It only wooks for HTTP protocol.

Sets the cookies maximum age in seconds.

Gets the Cookies maximum age in seconds, By default, -1 is returned, which indicates that the cookie will persist until browser shutdown.
